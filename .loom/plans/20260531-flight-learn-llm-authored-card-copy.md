## Summary

This plan repairs the `/flight-learn` comprehension path after operator feedback showed that the current focused card still exposes too much deterministic detector/provenance language. The intended model-enabled experience is now sharper: when the local model path is explicitly enabled and hard gates pass, the primary card copy should be local-LLM-authored across the reading surface, while evidence remains deterministic, hidden by default, and inspectable on demand.

This needs multiple tickets because it crosses a shared local-model response contract, validation/privacy gates, focused-card rendering, local-runtime quality evidence, and operator comprehension validation. Completion should leave `/flight-learn` with a more seamless user-facing card that can be validated before corpus/outcome collection resumes.

## Strategy

The operator's phrasing, “all of those fields should be generated by a local LLM,” is correct only if “generated” means **authored display copy from deterministic facts**. It is wrong if it means generated evidence, generated route decisions, generated expected behavior, or stored truth. This plan preserves that distinction.

The product direction is:

- local LLM output writes the primary explanation copy when explicitly enabled and gated;
- deterministic facts/evidence remain the source of truth;
- evidence/provenance is hidden by default and inspectable by expansion;
- `Raw clue` and `Why suggested` stop being primary sections;
- the replacement user-facing field is closer to `Why this was flagged`;
- expected behavior is never invented when unknown;
- route choices remain human-selected and candidate-only;
- deterministic fallback remains safe and non-dead-ending, but it does not need to be as rich as the model-enabled path.

The route is contract-first, then render, then evidence:

1. **Extend the local card-copy contract before UI changes depend on it.** The current response shape primarily covers headline, what happened, impact, and expected behavior. It does not provide a safe user-facing replacement for detector/provenance/evidence-summary copy. Add explicit fields and validators first.
2. **Render the new field jobs as the primary card.** The focused card should consume the gated card-copy result and demote evidence/provenance internals behind expansion. Fallback should avoid internal-debug default copy even when no model text is available.
3. **Measure the local runtime path separately from the UI shape.** Fake providers can prove behavior and safety, but real local-runtime usefulness remains a separate claim. Bonsai 4B is available from prior work, but any runtime invocation should be explicit in the ticket run.
4. **Rerun comprehension validation from the new card shape.** The old blocked review packet should not be used to open corpus collection. The operator should review the repaired render pack and answer what happened, why it matters, and route/observe/no-artifact decision.

Scope included:

- local-model response/fact-packet contract changes for card-copy fields;
- hard gates for all new fields;
- focused-card rendering changes that hide raw clue/provenance by default;
- fake-provider tests and render artifacts;
- local-runtime replay/smoke if explicitly authorized during execution;
- successor operator comprehension validation.

Scope excluded:

- hosted/non-loopback model calls, automatic downloads, telemetry, or new model families;
- default model enabling or extra top-level slash commands;
- route ranking, classifier automation, artifact application, rule/source/docs/Loom mutation, or storage truth mutation from model text;
- weakening accepted-narrative judge gates;
- using raw private sessions, raw local paths, prompts, transcripts, stack traces, or secrets in Loom artifacts.

Replan triggers:

- the broader field contract cannot be made fact-cited and privacy-safe;
- local model output repeatedly invents expected behavior, evidence, routes, or internal claims;
- UI integration requires model text to become stored truth or route authority;
- deterministic fallback can only avoid internals by hiding too much to be non-dead-ending;
- real local-runtime evidence shows the configured model cannot produce usable all-field card copy.

## Execution Units

### Unit: Local card-copy contract and hard gates

Ticket: `ticket:20260531-flight-learn-llm-card-copy-contract`

Extend the explicitly enabled local-model diagnosis/card-copy contract so it can author all primary focused-card explanation fields from bounded facts: problem/headline, what happened, why it matters, expected behavior wording, why this was flagged, and evidence summary. Add field-specific schema/gates so evidence remains source-of-truth and unsafe/unsupported/internal/debug output fails closed or falls back.

Scope boundary: local diagnosis model contract, adapter schemas if needed, focused tests, and evidence only. Do not change rendering behavior beyond what tests need to exercise the contract. Do not change model defaults, command surface, storage, routing, artifacts, source/docs/Loom mutation, or classifier behavior.

Order reason: rendering the new UX before the contract exists would force the UI to either keep detector strings or infer unsupported copy.

Validation/audit expectation: fake-provider tests for every field, rejected unsafe/internal/generated-evidence cases, typecheck/build/full tests as practical, privacy/side-effect scans, evidence dossier, and audit before closure.

### Unit: Focused-card rendering for authored copy and hidden provenance

Ticket: `ticket:20260531-flight-learn-llm-card-copy-rendering`
Depends On: `ticket:20260531-flight-learn-llm-card-copy-contract`

Update the focused `/flight-learn` card to use gated local card-copy fields as the primary reading surface when available. Replace default `Raw clue`/`Why suggested` sections with user-facing `Why this was flagged` copy, keep evidence collapsed by default, and keep route/edit/evidence/dismiss/skip/quit actions visible. Deterministic fallback should avoid putting detector names, cluster IDs, confidence scores, raw commands, or raw paths in the default primary path.

Scope boundary: focused-card renderer, command integration tests only where required to pass local-model display state through, render artifacts, and evidence. Do not add model calls by default, alter route choices, store model wording, or create/apply artifacts.

Order reason: this is the first user-visible repair but it depends on the new contract so the UI can stay fielded and fact-bounded.

Validation/audit expectation: render artifacts at representative widths for draft/validated/fallback states; focused tests proving hidden provenance, collapsed evidence, visible keys/actions, and no storage/routing side effects; privacy scan; audit focused on trust-boundary and cognitive-load regressions.

### Unit: Local-runtime card-copy replay and render pack

Ticket: `ticket:20260531-flight-learn-llm-card-copy-runtime-replay`
Depends On: `ticket:20260531-flight-learn-llm-card-copy-rendering`

Exercise the product card-copy path over a synthetic/redacted corpus and preserve a render pack showing whether the configured local runtime can produce usable all-field card copy. Fake providers may be used for deterministic behavior tests. Real Bonsai 4B/llama.cpp runtime evidence should be gathered only with explicit runtime authorization during execution; if not authorized or unavailable, record that as a blocker rather than claiming real-runtime usefulness.

Scope boundary: evidence/harness/artifact work and, if needed, tiny artifact-local render harnesses. No product source changes unless a separate implementation ticket is created. No raw private sessions or unredacted local artifacts in Loom.

Order reason: the UI can be implemented with fake providers, but comprehension validation and corpus collection need representative card output, ideally from the intended local runtime or an explicit blocker explaining why not.

Validation/audit expectation: metrics for parse/schema/gate pass, field coverage, fallback reasons, unsafe rejections, latency if real runtime runs, redacted render artifacts, privacy scan, and audit before using the result as a gate input.

### Unit: Successor operator comprehension validation

Ticket: `ticket:20260531-flight-learn-llm-card-copy-comprehension-validation`

This successor validation replaces the blocked `ticket:20260529-flight-learn-comprehension-validation` packet as the gate to dogfood corpus/outcome collection.

Scope boundary: evidence and validation artifacts only unless a tiny artifact-local harness is needed. Do not collect raw private sessions into Loom. Do not treat schema validity, render success, or local-runtime output as operator comprehension.

Order reason: corpus/outcome collection remains unsafe until the operator can actually interpret the repaired card.

Validation/audit expectation: operator notes for representative cards, 80% clear/mostly-clear threshold with zero safety/privacy blockers before proceeding, explicit fallback assessment, privacy scan, and audit before closure or dogfood unblocking.
