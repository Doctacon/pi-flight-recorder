## Residual Risk

- The corpus is synthetic/redacted and only 15 cases; it does not prove real-user usefulness or real private-session behavior.
- `exact-example-single-json` still has `0/15` accepted narratives and high timeout pressure (`11/15` timeout fallbacks; judge latency around `5001-5002ms`), so the improvement is only format/schema staging, not end-to-end narrative quality (`.loom/evidence/artifacts/20260529-bonsai-4b-schema-prompt-tuning/schema-prompt-tuning-summary.json:75-111`).
- The prior-baseline comparison is comparable enough for this ticket, but not statistically stable: the within-run `baseline-current` fallback distribution differs from the prior baseline (`7` timeout / `7` malformed / `1` schema-invalid versus prior `12` malformed / `2` schema-invalid / `1` timeout), so future work should avoid overinterpreting exact failure-category deltas (`.loom/evidence/artifacts/20260529-bonsai-4b-schema-prompt-tuning/schema-prompt-tuning-summary.json:31-72`; prior `.loom/evidence/artifacts/20260528-bonsai-4b-narrative-validation/real-bonsai-4b-narrative-summary.json:8-16`).
- The local judge was the same Bonsai 4B self-judge path and timed out for all verifier-passing candidates; this run does not answer whether an independent local judge, longer timeout, grammar-constrained JSON, or another local/open-source model would accept the candidates.

## Closure Recommendation

Close `ticket:20260529-bonsai-4b-schema-prompt-tuning` as reviewed/accepted for its evidence-only scope: prompt/schema tuning materially improved structured-output compliance for `exact-example-single-json`, all outputs still failed closed, no unsafe outputs were accepted, privacy/local-first posture is intact, and no productization claim is supported. Feed this result into `ticket:20260529-flight-recorder-core-loop-stocktake`; if model work continues, shape a separate judge/latency replay or grammar-constrained JSON investigation rather than continuing broad prompt tweaks inside this ticket.
