## Summary

Implement the CLI/library query surface for failure memory. The closure claim is that a user can sync fixture or local session data and run a query that returns ranked, evidence-backed prior failure episodes without Pi UI or embeddings.

## Scope

May change:

- CLI commands, library query API, result ranking/formatting, help text, structured JSON output, and CLI tests.

Must not change:

- No Pi extension command registration yet.
- No tmux/Ghostty UI.
- No hosted embeddings.
- No automatic code edits.

Suggested command names are implementation details, but the surface should support at least a sync/index action and a query action equivalent to “seen this before.” JSON output should exist so the later Pi extension can wrap the same core behavior.

Likely first Ralph run: implement CLI commands against fixture/temp data, text and JSON result formatting, friendly empty-state errors, and tests.

Stop if query results cannot cite source refs from storage/extraction; return to storage/extraction tickets instead of inventing uncited answers.

## Acceptance

- ACC-001: CLI can sync/index configured session fixtures or source directories into a local temp data directory.
  - Evidence: CLI integration test runs sync against fixtures and exits successfully.
  - Audit: Review the command does not write to real user data during tests.

- ACC-002: CLI query returns ranked results for exact error/command/file tokens without embeddings.
  - Evidence: CLI integration test queries a known fixture token and asserts expected episode appears.
  - Audit: Review ranking and limits are deterministic enough for user trust.

- ACC-003: Human-readable output separates prior failure, likely fix, failed attempts, evidence refs, confidence, and limits.
  - Evidence: Snapshot or structured output test.
  - Audit: Review output avoids generic “related sessions” language.

- ACC-004: JSON output exposes stable fields for a Pi wrapper to consume.
  - Evidence: Test parses JSON output and asserts schema fields.
  - Audit: Review fields do not expose unredacted secret-looking snippets.

- ACC-005: Empty index, no matches, and parse-warning states produce friendly actionable messages.
  - Evidence: CLI tests for empty/no-match/warning cases.
  - Audit: Review error text guides next steps without overclaiming.
